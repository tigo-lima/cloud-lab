# Terraform Notes

## Purpose
Recreate the manually built Azure infrastructure using Infrastructure as Code.

## Resources deployed
- Resource Group
- Virtual Network
- Subnet
- Network Security Group
- SSH and HTTP rules
- Public IP
- Network Interface
- Linux Virtual Machine

## Benefits observed
- Repeatable deployment
- Clear dependency handling
- Faster rebuild process
- Better documentation through code

## Key lesson
The Terraform version makes the environment reproducible and easier to maintain than manual CLI creation.
